# `index/papers.json`

The full master list of every paper + markscheme in `papers/`.
Generated by `src/index_papers.py` (Phase 1).

## Schema (v1)

```json
{
  "schema_version": 1,
  "generated_at_utc": "2026-06-13T20:33:00+00:00",
  "paper_count": 3,
  "papers": [
    {
      "slug": "aqa-84621h-chemistry-higher-2024-05",
      "board": "aqa",
      "spec": "8462",
      "paper": "1H",
      "subject": "chemistry",
      "tier": "higher",
      "exam_ym": "2024-05",
      "kvdb_bucket": "05c5c944-c9b1-57bb-b313-9cb59673d866",
      "files": {
        "qp": "AQA-84621H-QP-JUN24.PDF",
        "ms": "AQA-84621H-MS-JUN24.PDF"
      }
    }
  ]
}
```

## Fields

- `schema_version` — integer. Bump on breaking changes.
- `generated_at_utc` — when the indexer last ran (ISO 8601, UTC).
- `paper_count` — convenience count, equal to `papers.length`.
- `papers[].slug` — content-derived: `<board>-<spec><paper>-<subject>-<tier?>-<YYYY>-<MM>`.
- `papers[].board` — `aqa` / `edexcel` / `ocr` / `eduqas` / `wjec`.
- `papers[].spec` — the awarding-body spec code (e.g. `1MA1`, `8462`, `8702`).
- `papers[].paper` — the paper reference within the spec (e.g. `1H`, `1`).
- `papers[].subject` — slug-form subject (e.g. `chemistry`, `english-literature`).
- `papers[].tier` — `higher` / `foundation` / `null` if untiered.
- `papers[].exam_ym` — `YYYY-MM` of the exam sitting (the QP's date wins;
  AQA publishes its MS the month after the exam, which would otherwise
  split QP/MS for the same sitting into two groups).
- `papers[].kvdb_bucket` — UUIDv5 over `board|spec|paper|exam_ym`. Stable
  across re-runs of the indexer.
- `papers[].files.qp` / `files.ms` — the original filenames (no rename).

## Re-running

`src/index_papers.py --clean` removes the per-slug subdirectories and
this file, then re-creates them from scratch. Idempotent: re-running
produces the same `kvdb_bucket` and same `slug` for the same inputs.

## Idempotency check

A re-run today (2026-06-13 21:33) confirmed the meta files differ
*only* in `indexed_at_utc` / `generated_at_utc`. Slugs, bucket ids,
discovered board / spec / paper / tier / subject / exam date are
all byte-identical.
