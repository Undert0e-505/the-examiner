# pages/

**Phase 3+ — not built yet.** The folder is empty by design. This
README describes the design we have in mind for the GitHub Pages
site.

GitHub Pages root. Auto-deployed from `main` of this repo.

```
pages/
├── index.html                 # lists all assessments (newest first)
└── assessments/
    └── <batch>.html           # one per assessment (the student-feedback page)
```

The assessment page is generated by `src/publish.py`. The student
opens it, sees each mark, and clicks per-mark agree / disagree /
free text. Responses PUT to the paper's KVdb bucket.

The index page is a hand-written / Jimothy-maintained "dashboard"
listing all assessments with status badges (assessed / feedback received
/ corrections applied).
